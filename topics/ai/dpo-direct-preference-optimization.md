# DPO（Direct Preference Optimization）详解

## 摘要

DPO 是 2023 年由 Stanford 提出的颠覆性对齐方法。它通过一个优美的数学等价变换，将 RLHF 的三步复杂流程简化为**一个二分类交叉熵损失**。核心洞察是"你的语言模型本身就是隐含的奖励模型"——不需要单独训练 RM，也不需要 PPO。DPO 已成为开源社区最流行的对齐方法。

---

## 1. 从 RLHF 到 DPO —— 为什么需要简化

### 1.1 RLHF 的痛点

RLHF 虽然效果好，但管线极其复杂：

```
RLHF 需要维护的模型：
1. 策略模型 (π_RL)     — 正在被训练的模型
2. 参考模型 (π_SFT)    — 冻结的 SFT 模型（KL 基准）
3. 奖励模型 (RM)       — 单独训练的偏好评分器
4. 价值模型 (Critic)   — PPO 需要的状态价值估计

总计：4 个模型同时驻留 GPU → 显存噩梦
```

**工程挑战**：
- PPO 训练不稳定，经常崩溃
- 多模型协调复杂，调试困难
- 超参数空间巨大（PPO 的学习率、KL 系数、clip 范围、GAE λ、epoch 数...）

### 1.2 DPO 的承诺

> "你的语言模型本身就是隐含的奖励模型（Your Language Model is Secretly a Reward Model）"
> — DPO 论文副标题

DPO 用一个单一的分类损失替代了整个 RLHF 管道——不需要显式训练 RM，不需要 PPO，不需要 Critic。

---

## 2. 数学推导：从 RLHF 到 DPO 的等价变换

这是 DPO 最精彩的部分。理解这个推导链是理解 DPO 的关键。

### 2.1 起点：RLHF 的 KL 约束优化

RLHF 的目标是：
```
max_π E[r(x,y)] - β · D_KL(π || π_ref)
```

### 2.2 关键步骤 1：闭式最优解

这个 KL 约束的优化问题存在**闭式最优解**：

```
π_r(y|x) = (1/Z(x)) · π_ref(y|x) · exp(r(x,y)/β)
```

其中 `Z(x) = Σ_y π_ref(y|x) · exp(r(x,y)/β)` 是配分函数（归一化因子）。

**含义**：给定任意奖励函数 `r`，对应的最优策略就是把 `π_ref` 按 `exp(r/β)` 重新加权。

### 2.3 关键步骤 2：逆向表达 —— 策略隐式定义奖励

将上式重新排列，把奖励表达为策略的函数：

```
r(x,y) = β · log(π_r(y|x) / π_ref(y|x)) + β · log Z(x)
```

这就是"隐式奖励"——**奖励可以被策略概率比完全表达**。

### 2.4 关键步骤 3：代入 Bradley-Terry 偏好模型

偏好概率通常用 Bradley-Terry 模型建模：
```
P(y_w ≻ y_l | x) = σ(r(x, y_w) - r(x, y_l))
```

将步骤 2 的隐式奖励代入：
```
P(y_w ≻ y_l | x) = σ( β·log(π(y_w)/π_ref(y_w)) - β·log(π(y_l)/π_ref(y_l)) + β·log Z(x) - β·log Z(x) )
```

**配分函数 `Z(x)` 只依赖于 `x`，在差值中互相抵消！** 这就是 DPO 的关键洞察。

### 2.5 最终结果：DPO 损失函数

```
L_DPO(π_θ; π_ref) = -E_{(x,y_w,y_l)}[ log σ( β·log(π_θ(y_w|x)/π_ref(y_w|x)) - β·log(π_θ(y_l|x)/π_ref(y_l|x)) ) ]
```

**这个损失函数仅依赖于**：
1. 当前策略 `π_θ`（可训练）
2. 参考模型 `π_ref`（冻结）
3. 偏好数据 `(x, y_w, y_l)`

**不需要的**：
- ❌ 显式奖励模型 RM
- ❌ PPO / 在线采样
- ❌ Critic / 价值函数

## 3. DPO 的训练过程

### 3.1 数据准备

和 RLHF 的第二步完全相同：
- 同一个提示 `x`，生成两个回答
- 标注者选出更好的 `y_w` 和更差的 `y_l`
- 数据格式：`(x, y_w, y_l)` 三元组

### 3.2 训练步骤

```
1. 初始化 π_θ = π_ref（通常从 SFT 模型开始）
2. 冻结 π_ref
3. 对每个 batch 的偏好数据：
   a. 计算 π_θ 下 chosen 和 rejected 的对数概率
   b. 计算 π_ref 下 chosen 和 rejected 的对数概率
   c. 将两者的 log-ratio 差值通过 sigmoid
   d. 计算交叉熵损失
4. 反向传播更新 π_θ
```

### 3.3 代码规模对比

| | 代码量（Python） |
|---|---|
| 完整 RLHF（SFT + RM + PPO） | ~500 行 |
| DPO | ~20 行 |

## 4. DPO vs RLHF 深度对比

### 4.1 算法层面

| 维度 | DPO | RLHF (PPO) |
|------|-----|------------|
| **模型数量** | 2 个（策略 + 冻结参考） | 4 个（策略 + 参考 + RM + Critic） |
| **训练方式** | Off-policy（分类损失） | On-policy（策略采样） |
| **稳定性** | 极稳定（监督学习级） | 常不稳定（RL 级） |
| **超参数** | β 一个关键参数 | β + PPO 的 ~10 个超参数 |
| **数据来源** | 静态偏好数据 | 在线生成 + RM 打分 |
| **奖励模型** | 隐式（策略即 RM） | 显式训练，可被欺骗 |
| **训练时长** | 小时级 | 天级 |

### 4.2 核心权衡

**DPO 的优势**：
1. 简单到极致——任何会做监督学习的人都能实现
2. 训练极稳定——不会出现 PPO 的崩溃
3. 计算成本低——省去了 RM 和 Critic 的训练

**DPO 的劣势**：
1. **Off-policy**：只能从已有的静态偏好数据中学习，无法在线探索
2. **容易过拟合**：会快速拟合偏好数据集，而不是学到泛化能力
3. **依赖数据质量**：偏好数据的覆盖范围决定了模型的上限
4. **不适合 Agent 场景**：无法处理多轮交互的信用分配问题

### 4.3 什么时候用 DPO vs RLHF

| 场景 | 推荐方法 | 原因 |
|------|----------|------|
| 开源模型微调 | DPO | 简单、快速、成本低 |
| 偏好对齐 | DPO | 效果接近 RLHF |
| 需要在线探索 | RLHF | DPO 无法探索新行为 |
| 复杂奖励函数 | RLHF | RM 可以编码复杂偏好 |
| Agent 多轮训练 | GRPO/ARPO | DPO 不适合多轮 |
| 资源受限 | DPO | 不显存不够时唯一选择 |

## 5. DPO 的变体家族

DPO 的巨大成功催生了一个"DPO 家族"：

### 5.1 IPO（Identity Preference Optimization, 2023）

**问题**：DPO 对偏好数据中的噪声敏感，容易过拟合。

**方案**：添加正则化项，让 DPO 可以从**初始模型**（而非 SFT 后的模型）开始训练。

**损失函数**：
```
L_IPO = ((log(π_θ(y_w)/π_ref(y_w)) - log(π_θ(y_l)/π_ref(y_l)) - 1/(2β))²
```

### 5.2 KTO（Kahneman-Tversky Optimization, 2024）

**问题**：DPO 需要成对的偏好数据 `(chosen, rejected)`，标注成本高。

**方案**：基于前景理论（Prospect Theory），只需要 👍/👎 评分，不需要成对比较。

**灵感**：人类对损失的敏感度高于收益（损失厌恶）。

### 5.3 ORPO（Odds Ratio Preference Optimization, 2024）

**问题**：SFT + DPO 是两个独立阶段，数据利用不充分。

**方案**：**不需要参考模型**，将 SFT 损失和偏好优化合并为单一损失函数。

**核心**：用优势比（odds ratio）来建模偏好。

### 5.4 SimPO（Simple Preference Optimization, 2024）

**问题**：DPO 依赖参考模型增加了计算开销。

**方案**：**也不需要参考模型**，直接用长度归一化的对数概率作为奖励信号。

**额外收益**：天然控制生成长度（长度归一化防止模型偏好冗长输出）。

### 5.5 DPO 变体对比

| 变体 | 年份 | 需要参考模型 | 数据格式 | 核心创新 |
|------|------|-------------|----------|----------|
| **DPO** | 2023 | ✅ 需要 | 成对 (chosen, rejected) | 策略即 RM |
| **IPO** | 2023 | ✅ 需要 | 成对 | 正则化防止过拟合 |
| **KTO** | 2024 | ✅ 需要 | 单独 👍/👎 | 不需成对数据 |
| **ORPO** | 2024 | ❌ 不需要 | 成对 | SFT+对齐合并 |
| **SimPO** | 2024 | ❌ 不需要 | 成对 | 长度归一化 |

## 7. DPO 的局限性

### 7.1 理论局限

1. **隐式奖励的偏差**：
   - DPO 的隐式奖励 `β·log(π/π_ref)` 不一定是"好"的奖励函数
   - 它受限于策略模型的表达能力
   - 在低概率区域，隐式奖励的估计方差很大

2. **Off-policy 困境**：
   - DPO 只能从固定数据集中学习
   - 数据分布和最优策略分布不匹配时，DPO 效果受限
   - 这被称为"分布偏移"问题

### 7.2 实践局限

1. **容易过拟合**：DPO 可以在很少的 epoch 内完全拟合偏好数据
2. **需要大量偏好数据**：覆盖范围不够时效果差
3. **不适配 Agent 场景**：多轮工具调用的信用分配无法用 DPO 解决
4. **生成质量**：有研究指出 DPO 生成多样性可能低于 RLHF

## 8. DPO 在技术演进中的位置

```
RLHF (2017-2022)
  │  核心：RM + PPO → 对齐模型
  │  问题：管线和计算开销大
  │
  └── DPO (2023) ← 数学简化
       │  核心：策略即 RM，一个分类损失
       │  优势：简单、稳定、低成本
       │  问题：off-policy，无法在线探索
       │
       ├── DPO 变体 (2023-2024)
       │   ├── IPO：抗过拟合
       │   ├── KTO：不需成对数据
       │   ├── ORPO：不需参考模型
       │   └── SimPO：不需参考模型 + 长度归一化
       │
       ├── 回到 RL 路线
       │   └── GRPO (2024)：群组优势替代 Critic
       │       └── ARPO (2025)：多轮 Agent RL
       │
       └── DPO 的思想影响
           └── 隐式奖励的概念被后续方法广泛借鉴
```
