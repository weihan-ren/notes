# GRPO（Group Relative Policy Optimization）详解

## 摘要

GRPO 是 DeepSeek 在 2024 年提出的 PPO 简化变体。其核心创新是用**同一提示下多个输出的群组统计量**替代传统 PPO 中与策略等大的 Critic（价值函数）模型，将 RL 训练的内存开销减半。GRPO 最重要的应用是 DeepSeek-R1——通过纯 RL 训练让模型自发涌现出推理能力（"Aha Moment"）。

---

## 1. 起源：DeepSeekMath

GRPO 首次出现在 **DeepSeekMath 论文**（arXiv:2402.03300, Feb 2024）中。当时的目标是训练一个数学推理能力强的模型。

面对的挑战：PPO 需要 Critic 模型，而 Critic 通常和策略模型一样大——在训练大模型时，这意味着一倍的内存开销。

**GRPO 的解决方案**：如果 Critic 的作用只是提供一个"基准线"来计算优势，那为什么不直接用同一提示下的多个输出的统计量来替代？

---

## 2. 群组相对优势：为什么可以省掉 Critic

### 2.1 PPO 的做法

在 PPO 中，优势函数需要 Critic 来估计：
```
优势 = 实际奖励 - Critic 估计值
```

Critic 是一个与策略模型大小相当的神经网络，输出 `V(s)`（状态价值估计）。训练 Critic 需要额外的内存和计算。

### 2.2 GRPO 的做法

对每个提示 `x`，生成 G 个回答（如 G=64）：
```
优势_i = (r_i - mean(r_1...r_G)) / std(r_1...r_G)
```

核心思想：**群组归一化奖励（Group-Normalized Rewards）**

### 2.3 为什么这行得通

| PPO 的 Critic | GRPO 的群组统计量 |
|--------------|------------------|
| 学习一个函数来预测期望奖励 | 用 G 个实际奖励的均值来估计 |
| 需要额外的神经网络（与策略等大） | 不需要任何额外参数 |
| 估计精度随训练提升 | 精度随 G 增大而提升 |
| 可能学习缓慢或有偏 | 无偏估计（当 G 足够大时） |

**深层原因**：
1. LLM 的 RL 训练中，奖励只在最后一个 token 出现——Critic 需要学习逐 token 的价值，这本身就是困难的信用分配问题
2. RM 本身就用比较数据训练——群组相对优势与 RM 的训练方式天然对齐
3. G 可以很大（如 64-256），大幅度降低方差

## 3. GRPO 的优化目标

### 3.1 GRPO 损失函数

```
L_GRPO(θ) = E[ min( r_t(θ)·A_t , clip(r_t(θ), 1-ε, 1+ε)·A_t ) - β·D_KL(π_θ || π_ref) ]
```

与 PPO 的关键区别：
1. **优势 `A_t` 来自群组归一化**，而非 Critic 估计
2. **KL 惩罚直接加在损失中**，而非融入逐 token 奖励

### 3.2 KL 惩罚的处理

GRPO 在**损失函数层面**直接加 KL 惩罚：
- PPO：KL 惩罚融入逐 token 奖励中
- GRPO：KL 惩罚作为损失函数的一个独立项

**GRPO 的方式更简单**：
- 不需要在奖励计算中引入 KL 项
- 直接约束策略不偏离参考模型太远
- 实现约 10 行代码

---

## 4. GRPO vs PPO 深度对比

| 维度 | PPO | GRPO |
|------|-----|------|
| **Value Function** | 必须（等大的神经网络） | 消除（群组统计量替代） |
| **GPU 内存** | ~2× 策略模型 | ~1× 策略模型 |
| **优势估计** | GAE + 学习的 V(s) | 群组归一化奖励 |
| **KL 惩罚位置** | 融入逐 token 奖励 | 损失函数独立项 |
| **每次采样** | 1 个回答 | G 个回答（如 64） |
| **采样计算** | 低（1 次前向） | 高（G 次前向，需批处理） |
| **优势精度** | 依赖 Critic 训练质量 | 依赖 G 的大小 |
| **实现复杂度** | 高（Actor-Critic 架构） | 中（仅需策略模型） |
| **适用场景** | 通用 RL | 特别适合可验证奖励的 LLM 训练 |

## 5. DeepSeek-R1：GRPO 的革命性应用

### 5.1 DeepSeek-R1-Zero：纯 RL 推理涌现

**DeepSeek-R1-Zero** 的实验设计在当时是前所未有的：

> 直接将 GRPO 作用于基座模型（DeepSeek-V3-Base），**完全跳过 SFT！**

仅依靠规则化的可验证奖励（如数学题答案是否正确），模型自发涌现了：

1. **自我验证**："等一下，让我检查一下……"
2. **自我反思**："我重新考虑一下前面的步骤……"
3. **动态策略调整**：当一种方法不行时，自动切换
4. **长思维链自然增长**：模型自发学会"想得越久，做得越好"

> "通过纯强化学习，DeepSeek-R1-Zero 自然涌现了大量强大且有趣的推理行为。"
> — DeepSeek-R1 论文

**社区验证**：TinyZero 项目用不到 30 美元和 3B 模型就复现了这种"涌现"——纯 RL 确实能让模型学会推理。

### 5.2 为什么 GRPO 能催生推理涌现

1. **不需要 Critic 预测推理质量**：推理的正确与否是客观可验证的——不需要 Critic 来猜测"这一步好不好"
2. **群组比较天然适配数学推理**：同一题的不同解法可以直接比较
3. **大 G 值降低方差**：DeepSeek 使用极大的 G（推测 ≥64），优势估计非常精确
4. **RL 的探索性**：与 DPO 的静态数据不同，GRPO 在线生成多个回答，模型可以探索新的推理路径

### 5.3 `<think>` 推理标记机制

DeepSeek-R1 使用结构化输出格式：

```
<think>
让我逐步分析这个问题……
首先，我需要理解……
然后计算……
等等，我最好再检查一下……
</think>

答案是 42。
```

**`<think>` 的四个作用**：
1. **分离推理与输出**：模型在 `<think>` 内自由探索，最终答案简洁可验证
2. **RL 信用分配**：RM 只验证 `<answer>`，但奖励信号仍会塑造 `<think>` 内的推理质量
3. **防止奖励欺骗**：内部思考与评分答案解耦
4. **可解释性与蒸馏**：`<think>` 内容可检查、可提取用于训练小模型

## 6. DeepSeek-R1 完整训练流程

```
基座模型 (DeepSeek-V3-Base)
  │
  ├─ 冷启动 SFT（数千条高质量 CoT 示例）
  │    └─ 目的：解决 R1-Zero 的语言混杂和可读性问题
  │
  ├─ 推理导向 RL（GRPO，规则可验证奖励）
  │    └─ 重点：数学、编程、逻辑推理
  │
  ├─ 拒绝采样 SFT（~60 万正确推理轨迹 + 非推理数据）
  │    └─ 只保留答案正确的推理链
  │
  ├─ 全场景 RL（GRPO，规则奖励 + 人类偏好 RM）
  │    └─ 扩展到写作、翻译、问答等
  │
  └─ DeepSeek-R1
       │
       └─ 蒸馏到小模型（DeepSeek-R1-Distill 系列）
            └─ 1.5B / 7B / 8B / 14B / 32B / 70B
```

### R1-Zero vs R1 对比

| | R1-Zero（纯 RL） | R1（完整流程） |
|---|---|---|
| **SFT 先导** | 无 | 冷启动 SFT |
| **可读性** | 差（语言混杂、重复） | 好（流畅、清晰） |
| **RL 阶段** | 单阶段 | 两阶段（推理 RL + 全面 RL） |
| **拒绝采样** | 无 | ~60 万正确推理轨迹 |
| **非推理任务** | 未处理 | 写作、翻译、问答等 |

## 7. GRPO 的局限与权衡

| 优势 | 代价 |
|------|------|
| 无需 Critic（省一半内存） | 每次需要生成 G 个回答（计算量增加） |
| 实现更简单 | G 大小敏感：太小时优势估计噪声大 |
| 天然适配可验证奖励 | 使用学习的 RM 时可能放大 RM 偏差 |
| KL 直接在损失中 | 在 token 级 KL 估计可能噪声较大 |
| 推理能力自然涌现 | 回答长度可能爆炸（需余弦奖励/预算强制） |

### 7.1 长度控制问题

GRPO 训练中一个常见问题：模型学会写越来越长的回答，因为长回答在群组比较中可能获得更高奖励。

**解决方案**：
- **余弦奖励**：对超出期望长度的回答施加递减惩罚
- **预算强制**：直接截断超过最大长度的生成
- **长度归一化**：用回答长度除以奖励

## 8. GRPO 的演进：从单轮推理到多轮 Agent

### 8.1 GRPO 的定位

GRPO 主要针对**单轮推理任务**：
- 输入：一个数学题
- 过程：模型在 `<think>` 中推理
- 输出：答案
- 奖励：答案是否正确

### 8.2 从 GRPO 到 ARPO

ARPO（Agentic Reinforced Policy Optimization, 2025）将 GRPO 的思想扩展到多轮 Agent 场景：

- **GRPO 的局限**：假设单轮、静态任务，无法处理工具调用后的状态变化
- **ARPO 的创新**：熵驱动的自适应 Rollout——在工具调用后的高熵步骤上增加探索
- **共同根基**：都源自 PPO，都用群组/自适应采样来降低 Critic 的依赖

详见 [ARPO 专题笔记](arpo-agentic-reinforced-policy-optimization.md)。
